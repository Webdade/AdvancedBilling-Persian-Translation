# Advanced Billing For WHMCS — فارسی‌سازی (ناحیهٔ کاربری)

فارسی‌سازی افزونه **Advanced Billing For WHMCS** (ModulesGarden) با تمرکز بر **بخش‌های مربوط به مشتری (Client Area)**.

این ترجمه با هدف تجربهٔ کاربری روان و قابل‌فهم برای مشتریان فارسی‌زبان آماده شده است؛ پیام‌ها، دکمه‌ها، فرم‌ها و توضیحات سمت کاربر به **فارسی سلیس** برگردانده شده‌اند.

## محتوا

- `persian.php` — فایل زبان فارسی به فرمت PHP (`$_LANG[...]`) شامل تمام کلیدهای افزونه.
- `persian.json` — همان رشته‌ها به فرمت **JSON تخت** (`{"key":"value"}`) برای استفاده در بخش **Import Translation** ماژول (که فقط فرمت JSON را می‌پذیرد).

> هر دو فایل از یک منبع تولید شده‌اند و کلیدها کاملاً دست‌نخورده مانده‌اند؛ فقط **مقادیر** تغییر کرده‌اند.

## چه چیزی ترجمه شده و چه چیزی نه؟

- ✅ **ترجمه‌شده:** بخش‌های ناحیهٔ کاربری — ارتقای خودکار سرویس (Auto Upgrade)، یادآورها (Reminders)، رکوردها و جزئیات مصرف (Usage Records)، نمای کلی صورت‌حساب و خروجی CSV، شارژ حساب (Top up)، نمودار مصرف، لغو سرویس، اعلان‌ها، پیام‌های اعتبارسنجی فرم‌ها، وضعیت سرویس‌ها، اقلام منابع روی فاکتور (پهنای باند، فضای ذخیره‌سازی، مصرف پردازنده و ...) و پیام‌های عمومی رابط کاربری.
- ⏭️ **ترجمه‌نشده (عمداً):** بخش‌های **مدیریتی (Admin Area)** و ابزارهای داخلی ماژول (Access Control، Logs، Theme، Easy Translator، Requirements Checker، Queue). این بخش‌ها برای مدیر سیستم هستند و به انگلیسی باقی مانده‌اند.
- 🔢 **بدون تغییر:** واحدهای فنی استاندارد (`GB`, `MB/hr`, `MHz`, `GiB`, `IPv4`, `vCPU` و ...) که در همه‌جا یکسان و قابل‌فهم‌اند.

## نصب

**روش ۱ — فایل PHP:** فایل `persian.php` را در مسیر پوشهٔ زبان افزونه قرار دهید (همان مسیری که `english.php` در آن قرار دارد).

**روش ۲ — Import در ماژول:** از مسیر افزونه → Translations → Import Translation فایل `persian.json` را بارگذاری کرده و زبان مقصد (فارسی) را انتخاب کنید.

## نکات

- متغیرهای جایگزینی مانند `:name`، `:usage`، `:unit`، `:total`، `:min`، `:max` و ... دست‌نخورده باقی مانده‌اند و نباید تغییر کنند.
- در صورت به‌روزرسانی افزونه و افزوده‌شدن کلیدهای جدید، آن کلیدها تا زمان ترجمه به‌صورت انگلیسی نمایش داده می‌شوند.
